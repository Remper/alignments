Abstract

Together with [FOAF](http://xmlns.com/foaf/0.1/) and [Dublin Core Terms (DCTERMS)](http://purl.org/dc/terms/), the present [SocialLink](http://sociallink.futuro.media/) vocabulary provides terms to represent the alignments between entities of knowledge bases such as [DBpedia](http://dbpedia.org/) and accounts of social media such as [Twitter](http://twitter.com/), covering both the alignment links and all intermediate candidate alignment data. This vocabulary is used in the [SocialLink dataset](http://sociallink.futuro.media/index.html#download).

Introduction

The [SocialLink](http://sociallink.futuro.media/) vocabulary is intended to be used together with [FOAF](http://xmlns.com/foaf/0.1/) and [DCTERMS](http://purl.org/dc/terms/) to represent alignment data between knowledge base entities and social media accounts, as exemplified in the figure (SocialLink vocabulary concepts shown in red). In the example, entity [wikidata:Q76](http://wikidata.dbpedia.org/resource/Q76) -- standing for "Barack Obama" and here represented with all its DBpedia [owl:sameAs](http://www.w3.org/2002/07/owl#sameAs) aliases (e.g., [dbr:BarackObama](http://dbpedia.org/resource/BarackObama)) -- is aligned to account [twitter:BarackObama](http://twitter.com/BarackObama) (a [foaf:OnlineAccount](http://xmlns.com/foaf/0.1/OnlineAccount)) via property [foaf:account](http://xmlns.com/foaf/0.1/account). As shown, FOAF already provides concepts and properties for modeling the alignment, including [foaf:accountName](http://xmlns.com/foaf/0.1/accountName) representing the account username that we complement (in the [SocialLink dataset](http://sociallink.futuro.media/index.html#download)) with [dcterms:identifier](http://purl.org/dc/terms/identifier) to encode other account identifiers like the numerical user identifier used in Twitter APIs. To represent the candidate alignment data between entities and accounts produced during the generation of the alignments (which enables third parties to analyze/improve the alignment process), the SocialLink vocabulary introduces an additional [sl:Candidate](http://sociallink.futuro.media/ontology#Candidate) class representing the reified "candidate" alignment relation between an entity and an account. This means that each [sl:Candidate](http://sociallink.futuro.media/ontology#Candidate) instance represent a pair <entity, account> identified, respectively, with properties [sl:candidate](http://sociallink.futuro.media/ontology#candidate) and [sl:account](http://sociallink.futuro.media/ontology#account). An additional property [sl:confidence](http://sociallink.futuro.media/ontology#confidence) provides the confidence score associated to the candidate alignment (the one with the best score might be selected for the [foaf:account](http://xmlns.com/foaf/0.1/account) alignment by the alignment algorithm). Property [sl:rank](http://sociallink.futuro.media/ontology#rank) represents the position of the candidate in the ranked list of candidates for an entity, and as such can be used to speed up queries for the top candidate of an entity (independently of the existence of an alignment).

Classes

  • Candidate

Candidatec back to ToC or Class ToC

IRI: http://sociallink.futuro.media/ontology#Candidate

The reified 'candidate' relation between a knowledge base entity (identified by property [sl:candidate](http://sociallink.futuro.media/ontology#candidate)) and a social media account (identified by property [sl:account](http://sociallink.futuro.media/ontology#account)). Annotated with the confidence score of the candidate (property [sl:confidence](http://sociallink.futuro.media/ontology#confidence)) and its rank in the list of candidates for the entity (property [sl:rank](http://sociallink.futuro.media/ontology#rank)).
